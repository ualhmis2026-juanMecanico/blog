Configuración del Entorno de Desarrollo

Fecha: 14 de marzo de 2026
Autor: Cristobal Cano Quesada

Introducción

Para desarrollar nuestro blog de equipo con Hugo, hemos seleccionado Visual Studio Code (VSC) como nuestro editor principal. En esta entrada explico los pasos que hemos seguido para dejarlo a punto.

Pasos para la configuración

  1. Instalación de VSC: Descargamos e instalamos el editor para nuestra plataforma.
  2. Instalación de Hugo: Configuramos el binario de Hugo en el sistema para poder usar el comando hugo server desde la terminal integrada.
  3. Extensiones recomendadas: Instalamos extensiones para el resaltado de sintaxis de Markdown y TOML, facilitando la edición de archivos de contenido y configuración.
  4. Control de Versiones: Vinculamos VSC con nuestra cuenta de GitHub para gestionar las ramas y los Pull Requests sin salir del editor.

Beneficios

Tener un entorno bien configurado nos permite trabajar de forma colaborativa mediante el modelo GitHub Flow, facilitando la creación de feature branches y la integración de cambios en la rama principal.
